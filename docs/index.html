How to export

Get your .pen file into Figma in 4 steps


1

Open your .pen file

Open your design in Pencil. Make sure the file you want to export is the active editor.

2

Export with Claude Code (recommended)

If you have Claude Code with the Pencil MCP server connected, paste this prompt:

Export my currently open .pen file to JSON for the Figma importer plugin. Use batch_get with readDepth 10 to get all nodes, and get_variables to get all design tokens. Combine them into a single JSON file: { "variables": { ...variables... }, "children": [ ...nodes... ] } Save it as [filename]-export.json next to the .pen file.

Claude will read the full node tree and variables, then save a ready-to-import JSON file.

3

Alternative: Manual MCP export

If you prefer to run the MCP calls yourself, use these two commands and combine the results:

Get all nodes
batch_get(filePath: "your-file.pen", readDepth: 10)
Get variables
get_variables(filePath: "your-file.pen")
Combine into one JSON
{ "variables": { <result from get_variables> }, "children": [ <result from batch_get> ] }
4

Import into Figma

Open any Figma file, run the Pen to Figma Importer plugin, and drag your JSON file into the drop zone. Components are placed off to the right (x=4000), page frames at their original positions.

Prompt reference

Copy-paste prompts for common exports


Full export

Export my .pen file to JSON for the Figma importer. Include all nodes (readDepth: 10) and all variables. Save as export.json.

Components only

Export only the reusable components from my .pen file. Use batch_get with patterns: [{reusable: true}] and readDepth: 5. Include variables. Save as components-export.json.

Specific screens

Export the "Homepage" and "Dashboard" frames from my .pen file for the Figma importer. Include their full node trees and all variables.

Large file (chunked)

My .pen file is large. Export it for Figma by reading each top-level frame individually with readDepth: 8, then combining all results into one JSON with variables.

Good to know

Limitations & FAQ


Image fills

Image fills in .pen become grey placeholder rectangles in Figma. Re-add your images manually after import — the frames are sized correctly.

Mesh gradients

Figma doesn't support mesh gradients. They're approximated using the first color in the mesh at 50% opacity.

Icon fonts

Icon font nodes (Lucide, Material Symbols, etc.) render as text with the icon name. Install the icon font locally for correct rendering.

Fonts

Make sure fonts used in your .pen file are installed on your machine. The plugin falls back to Inter Regular if a font can't be loaded.

Large files

If batch_get returns too much data (>200k chars), export screens individually and combine them. Use the "Large file" prompt above.

Instance overrides

Property overrides (text, colors, sizing) are fully supported. Full subtree replacement in instances is not yet supported.
